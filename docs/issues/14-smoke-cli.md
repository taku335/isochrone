## ゴール

実フィード生成データに対して探索を実行する CLI を作り、結果の妥当性確認と性能計測(目標 <200ms)を行う。

## 作業内容

- `raptor-cli <stop名> <日付> <時刻>` → 到達停留所数、代表停留所への到着時刻、実行時間を出力
- 代表ケース(例: 栄 平日 8:00 / 名古屋駅 土曜 23:30 / 深夜帯 0:30)の到着分を、公式の市バス検索・時刻表と手動照合してスナップショット化
- 性能計測: ウォームアップ後 10 回の中央値を出力。<200ms を確認(未達なら原因をプロファイルし Issue 化)
- CI ではミニフィクスチャのみ(実フィードはローカル / 手動ワークフロー)

## 受入基準

- [ ] 代表ケースのスナップショットが公式情報と整合
- [ ] one-to-all 中央値 <200ms(実測値を Issue コメントに記録)
- [ ] 明らかな異常(未到達だらけ / 全停留所 0 分)を検知するサニティチェック内蔵

## 検証方法

CLI 実行ログ + スナップショットテスト。
