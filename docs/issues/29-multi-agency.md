## 概要(プレースホルダ — 実装計画なし)

名古屋市営地下鉄・他事業者の GTFS を統合し、実態に近い到達圏を描く。

## メモ

- `config/agencies.json` にエントリ追加 + 事業者 ID プレフィックスで衝突回避(設計済み)
- 課題: データサイズ予算(≤1.5MB)の再検討、事業者間徒歩乗換(駅⇔バス停)、運賃・改札考慮の要否
- 「市バスのみ」注記(#21)の文言更新を忘れないこと

## 実装準備

- 正規化済みの複数フィードを決定的な順序で統合し、事業者 ID・ID プレフィックス・エンティティ ID の衝突を拒否する
- 統合後の全停留所から徒歩接続を再生成し、事業者をまたぐ駅・停留所間の接続を作る
- manifest にソース別の表示名、feed version、有効期間、出典・ライセンスを保持する
- 全ソースが揃う共通有効期間だけを統合データの有効期間として公開する
- `dataset` / `validate` CLI で複数のキャッシュ済みフィードと明示的な複合 dataset ID を受け取り、同じ生成・検証経路を実行する

本番データへの適用、サイズ予算の決定、表示上の「市バスのみ」注記の変更は、公式かつ再配布可能な追加 GTFS の公開まで保留する。
