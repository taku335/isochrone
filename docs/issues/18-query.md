## ゴール

日付・出発時刻を指定して Worker 経由で探索を実行し、まず「到達停留所のドット表示」でエンジン出力を可視化する(ポリゴンより先にエンジンの正しさを確認できる状態を作る)。

## 作業内容

- 日付 + 時刻入力(既定: 今 / 深夜 0〜2 時台は前日サービス日の扱いをユーザーに分かるよう表示)
- 適用ダイヤ種別(平日 / 土曜 / 日休 等、#11 の出力)を UI に表示
- 探索実行 → 到達停留所を残余時間で色分けしたドットレイヤ表示(30 分 / 60 分)
- このドット表示は以降もデバッグモード(URL パラメータ)として残す

## 受入基準

- [ ] 代表ケースのドット表示が #14 の CLI 出力と一致
- [ ] 深夜時刻指定でも正しいサービス日で探索される
- [ ] 適用ダイヤ種別が表示される

## 検証方法

CLI 出力との突合(停留所数と代表停留所の到着分)。
